Compiling info

To compile the book code provided on this CD you'll need to do a few things.

1. Copy files off the CD

The first step is to copy the entire directory tree (starting in this directory) to your hard drive.  You can exclude certain chapters if you like, but the DevStudio Workspace (dsw) files in the "chapter xx" directories are counting on all the files below them being there. 

2. Install DirectX 8.0a

Next you need to make sure DirectX 8.0 or above is installed on your machine.  For your convenience the 8.0a SDK is included on this CD.  You should also make sure you have the newest version of your video card drivers installed.  Check their web site.

For sanity's sake, you should install the debug run-time of the DirectX dll's.  They provide lots of error checking to help you find bugs.

3. Set up DirectX in Visual Studio

Visual Studio ships with the headers for DirectX 3.0.  You need to do some work to make sure that the compiler can find the 8.0 headers (for compiling) and 8.0 libraries (for linking).  To do this, go to the tools->options area, and click on the 'directories' tab.  Set up things to look like they do in the following two images (of course, using the directory where you installed the SDK... on my machine it's in d:\mssdk).

The important thing is to make sure the DirectX 8 files are at the top of the list (by default they'll be put at the bottom).  When the compiler needs a header (like d3d8.h) it searches the list from top to bottom.  It needs to search the DX8 directory (with the 8.0 version of the header) before it searches the VC include directories (which have the 3.0 version of the D3D header).

4. Getting new projects to compile

The above 3 steps should be enough to make the provided chapters and projects compile correctly.  If you want to create new projects using the provided code, you can do the following:

If the workspace you're using has the libraries made in the game (GameLib, Math3d, and NetLib, though you may not need all three), the best thing to do is to set up dependencies for those projects (in project->Dependencies...).  For example, here is the dependency dialog for the GameClient project.

The other thing you'll need to do is link all the right libraries (in Project->Settings...).  Any project using GameLib needs the following libraries linked:

    winmm.lib d3d8.lib dxguid.lib dinput8.lib dsound.lib d3dx8.lib

Any project using NetLib needs the winsock DLL:

    ws2_32.lib

Finally, if the libraries aren't in the workspace and you just have the actual .lib files sitting in the directory, you need to link them in too. 

Happy coding.  If you have any trouble, help is available on the web at www.wordware.com.
